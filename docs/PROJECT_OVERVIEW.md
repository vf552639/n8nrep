# SEO CONTENT GENERATOR

## Описание

Standalone веб-приложение на Python для массовой автогенерации SEO-оптимизированных статей в нише iGaming. Полностью заменяет предыдущий n8n-воркфлоу. Система принимает ключевое слово + ГЕО + язык и автоматически проходит полный цикл: сбор SERP, парсинг конкурентов, анализ структуры через 4 LLM-агента, генерация контента, мультиэтапная редактура (сравнение с конкурентами, мнение читателя, улучшения, финальная правка), факт-чекинг, генерация HTML-страницы по шаблону сайта, мета-теги — и сохранение готового результата.

Поддерживается как генерация одиночных статей, так и массовая генерация целых сайтов через систему проектов с блупринтами (шаблоны типов сайтов с набором страниц). Для каждой страницы блупринта задаётся **pipeline preset** (`full`, `category`, `about`, `legal`, `custom`) и при необходимости свой список шагов; выполнение идёт по плану **`_pipeline_plan`** (см. **`app/services/pipeline_presets.py`**, **`docs/CURRENT_STATUS.md`**, 6 апреля 2026).

## Цель проекта

Автоматизировать создание SEO-контента для 10+ iGaming-сайтов в разных ГЕО и на разных языках. Убрать зависимость от n8n cloud (сбои, лимиты, стоимость), получить полный контроль над пайплайном, промптами и масштабированием.

## Целевая аудитория

- SEO-команда (основные пользователи админ-панели)
- Контент-менеджеры (проверка и публикация статей)
- Разработчик/DevOps (деплой и поддержка)

## Ключевые функции

1. Pipeline генерации с **настраиваемым набором шагов** по странице блупринта (пресет + optional custom); полный «классический» путь включает SERP → Scraping → четыре этапа анализа → fact-check структуры → генерация (в т.ч. отдельные агенты **about** / **legal**) → сравнение → reader opinion → interlink → improver → final edit → content fact-check → HTML → meta + опциональная цепочка image generation
2. Система проектов с блупринтами — генерация целого сайта из N страниц одной кнопкой
3. Промпты хранятся в БД с версионированием, редактируются из UI без деплоя; для **Model Settings** в таблице **`prompts`** хранятся явные булевы флаги **`max_tokens_enabled`**, **`temperature_enabled`**, **`frequency_penalty_enabled`**, **`presence_penalty_enabled`**, **`top_p_enabled`** — UI и pipeline не «угадывают» включённость по числам; **`max_tokens`**, **`frequency_penalty`**, **`presence_penalty`**, **`top_p`** в HTTP-запрос к OpenRouter включаются только при **`true`** (при **`false`** ключи не отправляются — без явного **`top_p=1.0`** / нулевых штрафов); **`temperature`** передаётся всегда (**0.7** при выключенном тоггле). См. **`app/services/prompt_llm_kwargs.py`**, **`app/services/llm.py`**, миграция **`k5m6n7o8p9qb`**, **`CURRENT_STATUS.md`** (апрель 2026, **8.04.2026**). При сохранении промптов текст санитизируется от нестандартных разделителей строк (U+2028/U+2029, NBSP, BOM) — см. `CURRENT_STATUS.md`, март 2026
4. Дедупликация контента между страницами одного проекта (ProjectContentAnchor)
5. Exclude Words — глобальные и per-author запрещённые слова с автоматической валидацией и retry
6. Факт-чекинг сгенерированного контента (soft/strict режимы)
7. Перегенерация отдельных шагов с feedback и каскадной инвалидацией
8. Telegram-уведомления о success/failed задачах
9. Последовательный и параллельный режимы запуска задач
10. Массовый импорт задач через CSV
11. SERP viewer с экспортом в CSV/ZIP
12. Test Mode — пауза после primary_generation для ручного одобрения
13. **SERP URL review** — для **одиночных** задач (не для **`auto_mode`** проектов) после шага SERP пайплайн останавливается со статусом **`paused`** и **`_pipeline_pause.reason = serp_review`**: в UI можно удалить/добавить URL конкурентов, затем **`POST /api/tasks/{id}/approve-serp-urls`** возобновляет Celery-пайплайн; **`phase_scraping`** по-прежнему читает URL из **`task.serp_data["urls"]`**; миграция **`m9n0o1p2q3re`**. См. **`docs/CURRENT_STATUS.md`**, **13.04.2026**
14. **Автопарсинг meta для вручную добавленных URL** — в SERP review UI при добавлении URL сразу создаётся строка, затем асинхронно вызывается **`POST /api/tasks/fetch-url-meta`** и в строку подставляются `title`/`description`/`domain`; доступна per-row кнопка refresh meta. На backend `parse_html()` извлекает `meta_title`/`meta_description`, а `scrape_urls()` возвращает агрегаты `scraped_titles`/`scraped_descriptions` для fallback. См. **`docs/CURRENT_STATUS.md`**, **14.04.2026**
15. Опциональная генерация изображений (Midjourney через GoAPI + ImgBB) с паузой ревью в UI и инъекцией в HTML отдельным шагом `image_inject`; LLM-шаг формирует промпты по каждому MULTIMEDIA-блоку; извлечение блоков из outline — **мультиязычные ключи** (напр. `МУЛЬТИМЕДИА`, `MULTIMÉDIA`), строка/список в JSON, встройки в текст полей и **fallback** по сырому тексту outline, если в JSON не найдено (`app/services/image_utils.py`). После `html_structure` вставка выполняется по HTML-комментариям `<!-- MEDIA: ... -->` в порядке появления, с очисткой оставшихся комментариев (`docs/CURRENT_STATUS.md`, 15.04.2026; 28.03.2026)
16. Временное Redis-кэширование SERP и scraping: повторные запуски используют кэш и уменьшают расход внешних API
17. Полный JSON **meta_generation** сохраняется в статье как **`meta_data`** и отображается на вкладке **Metadata**; при формате **`{"results": [{Title, Description, H1, Trigger}, …]}`** в **`title`/`description` статьи** подставляется **первый вариант**; на шагах pipeline с трансформацией HTML — **подсчёт слов контента** и предупреждение при потере **> 7%** на шаге `html_structure`; для снижения потери контента — **`max_tokens`** в LLM, рекомендуемая модель/лимиты в seed, один recovery-вызов при сильной потере, при необходимости программная вставка HTML через **`programmatic_html_insert`** (`app/services/html_inserter.py`) — см. `CURRENT_STATUS.md`, март–апрель 2026

## Рабочая область промптов (UI)

Страница **`/prompts`** в приложении называется **«SEO Workflow Optimizer»**: трёхколоночный layout (список агентов → редакторы system/user → Variable Explorer), сверху панель **Model Settings** (градиентный toolbar в стиле macOS: **`ModelSelector`**, Max. Tokens / Temperature / Freq. / Pres. / Top P с **`ToggleSwitch`** и слайдерами, кнопка **Save** синяя с dirty-индикатором), **Test** в шапке карточки агента. Сохранение настроек агента — **`PUT /api/prompts/{id}`** с полями **`_*_enabled`** (обновление строки в БД **без** новой версии); тест промпта может слать несохранённые параметры через **`PromptTestLlmOptions`**. Версия из истории — отдельно через **Restore**. Гидратация формы: **`syncedPromptIdRef`** + **`[derivedActiveId, fullPrompt?.id]`** (refetch с тем же id не затирает локальные правки). В Variable Explorer есть в т.ч. **`site_template_html`**, **`site_template_name`**, **`legal_reference`**, **`legal_reference_html`**, **`legal_reference_format`**, **`legal_template_notes`**, **`page_type_label`**, **`legal_variables`**. Monaco: **`unusualLineTerminators: off`**. Выпадающий список моделей рендерится в **`document.body`** (портал + `position: fixed`). Глобальный **Header** на этой странице скрыт (`MainLayout`). Актуальные детали — **`docs/CURRENT_STATUS.md`**, **апрель 2026** (флаги **`_*_enabled`**) и **3 апреля 2026** (PUT, ModelSelector).

**Шаблоны сайта и legal:** управление HTML-оболочками — раздел **Templates** (`/templates`); привязка шаблона к сайту и JSON **`legal_info`** — **Sites** (`/sites`, деталь `/sites/:id`). Образцы legal-страниц — **Legal Pages** (`/legal-pages`); у записи **`legal_page_templates`** для подписи в UI используется только **`name`** (поле **`title`** удалено, **18.04.2026**); контент образца — **`content`** + **`content_format`** (**`text`** / **`html`**), опционально **`notes`** (**21.04.2026** — см. **`CURRENT_STATUS.md`**). **Двухуровневая привязка legal reference (апрель 2026, 18.04):** на странице блупринта с **`page_type` ∈ legal** можно задать **`default_legal_template_id`** (фолбек для проектов); в проекте по-прежнему **`legal_template_map`** — override; в пайплайне приоритет **Project → Blueprint default → пусто** (`inject_legal_template_vars`, **`GET /legal-pages/for-blueprint/{id}`** отдаёт **`default_template_id`** для автоподстановки в форме создания проекта). См. **`docs/CURRENT_STATUS.md`**, **18 апреля 2026** и **21 апреля 2026**.

**Задачи и pipeline в UI:** на списке задач — чекбоксы для `pending`, фильтр статуса в т.ч. **`paused`** (пауза SERP URL review), запуск выбранных через **`POST /api/tasks/start-selected`** (Celery chain). На **Projects** — чекбоксы и массовое удаление через **`POST /api/projects/delete-selected`** (в т.ч. **`force`** после **`skipped`**, **19.04.2026**); в модалке создания проекта — опциональные **Competitor URLs** (до **50**, мерж с SERP в **`phase_serp`**, **21.04 — task41**); на **Project Detail** — read-only список сохранённых URL. На странице задачи — табы pipeline/execution logs (+ **SERP URL review** при **`paused` + serp_review**, image/article review по состоянию); **Article Review** — Monaco HTML + Preview, опционально **`PUT .../step-result`**; шаги **SERP** и **scraping** — таблицы и метрики, LLM-шаги — результат / промпты / переменные (см. **`docs/CURRENT_STATUS.md`**, **13.04.2026**, «Задачи, Task Detail», **«Monaco для HTML»**).

**Боковое меню:** сворачивается в узкую колонку с иконками, состояние запоминается в `localStorage` (`sidebar_collapsed`).

1. Описание проблем
Текущий интерфейс имеет ряд визуальных проблем, которые затрудняют работу с админкой. Ниже перечислены все выявленные дефекты.

1.1. Поля ввода сливаются (все вкладки)
Streamlit по умолчанию не добавляет достаточный вертикальный отступ (margin-bottom) между элементами форм. В результате:
•	Лейбл следующего поля визуально «прилипает» к инпуту предыдущего поля
•	При двух колонках (st.columns) поля в левой и правой колонке сливаются — нет горизонтального разделителя
•	Особенно заметно на вкладке «Настройки» (скриншот прилагается) и в форме создания задачи (Страна / Язык)

1.2. Вкладка «Настройки» — конкретные дефекты
•	Заголовки «LLM & SEO APIs» и «Остальное» — разный визуальный вес (subheader vs subheader, но без разделителя снизу)
•	Между двумя колонками нет визуального разделителя — левая и правая группа полей «склеиваются»
•	Поле «Exclude Words» — используется text_input, но содержимое длинное (30+ слов). Текст обрезается, не видно полностью
•	Кнопка «Сохранить настройки» — маленькая, без акцента, прижата к левому краю
•	Нет отступов между полями — форма выглядит «слепленной»

1.3. Вкладка «Задачи» — Страна и Язык
В форме создания задачи поля «Страна» и «Язык» расположены в двух колонках (st.columns(2)) без gap. Selectbox-ы визуально сливаются друг с другом.

1.4. Вкладка «Авторы» — двухколоночная форма
Аналогичная проблема: поля «Имя» / «Страна», «Язык» / «Город» в st.columns(2) без gap.

1.5. Вкладка «Промпты» — параметры модели
Четыре колонки (Temperature / Freq Penalty / Pres Penalty / Top P) в st.columns(4) без gap — чекбоксы и number_input-ы тоже сливаются.
 
2. Что нужно сделать
2.1. Глобальный CSS-блок
Добавить блок st.markdown(<style>...) сразу после st.set_page_config() в начале файла frontend/app.py. Этот блок должен решать проблему сливающихся полей глобально, на всех вкладках.

Что должен содержать CSS:
CSS-селектор / свойство	Что делает и зачем
div[data-testid="stTextInput"], div[data-testid="stSelectbox"], div[data-testid="stTextArea"], div[data-testid="stNumberInput"]	margin-bottom: 12px — добавляет вертикальный отступ между ВСЕМИ полями ввода на ВСЕХ вкладках. Это главный фикс проблемы «слипания».
div[data-testid="column"]	padding-right: 12px; padding-left: 4px — отступы внутри колонок, чтобы поля не прилипали к краям.
Первая колонка в форме Настроек	border-right: 1px solid rgba(0,0,0,0.06) — тонкая вертикальная линия-разделитель между левой и правой группами полей.
Заголовки h3 внутри колонок	border-bottom: 2px solid rgba(0,0,0,0.06); padding-bottom: 8px — подчёркивание заголовков секций.
Лейблы полей (label)	margin-top: 4px; font-weight: 500 — чуть жирнее и с отступом сверху для читаемости.
Кнопка формы (submit)	margin-top: 16px — отступ сверху, чтобы кнопка не прилипала к последнему полю.
Метрики на дашборде (metric-container)	background-color: #f8f9fb; border: 1px solid #e8eaed; border-radius: 10px — карточки-плитки вместо голых цифр.

Важно: CSS добавляется через st.markdown(«<style>...</style>», unsafe_allow_html=True). Все data-testid-селекторы — стандартные для Streamlit 1.31+. При обновлении Streamlit нужно проверить, что тест-айди не изменились.
 
2.2. Вкладка «Настройки»
Точечные правки в коде вкладки (tabs[6]):

Элемент	Сейчас	Нужно
Колонки	st.columns(2)	st.columns(2, gap="large") — добавляет большой горизонтальный отступ между колонками
Заголовки секций	st.subheader("LLM & SEO APIs") и st.subheader("Остальное")	st.markdown("### 🔑 LLM & SEO APIs") и st.markdown("### 🛠️ Системные и Остальные") — одинаковый размер, с эмоджи, + CSS-подчёркивание через h3 стиль
Exclude Words	st.text_input(...)	st.text_area("Слова-исключения (глобально)", height=100, ...) — многострочное поле, видно весь список слов
Кнопка	st.form_submit_button("Сохранить настройки")	st.form_submit_button("💾 Сохранить настройки", use_container_width=True) — на всю ширину, с иконкой
Info-блок	st.info("Здесь хранятся ключи...")	Заменить на st.caption("API ключи и параметры. Оставьте поле пустым, чтобы не менять значение.") — компактнее, без синей плашки
Заголовок вкладки	st.header("Настройки системы")	st.header("⚙️ Настройки системы") — добавить эмоджи

Результат: Два визуально разделённых блока (с тонкой линией между ними), поле Exclude Words видно целиком, кнопка заметная.
 
2.3. Вкладка «Задачи»
В форме «Создать новую задачу» (внутри st.expander):

Элемент	Сейчас	Нужно
Страна / Язык	col_geo, col_lang = st.columns(2)	st.columns(2, gap="medium") — отступ между двумя selectbox-ами

Одно изменение — одна строка. Но визуально поля перестанут сливаться.

2.4. Вкладка «Авторы»
В форме «Создать нового Автора» (внутри st.expander → st.form):

Элемент	Сейчас	Нужно
Имя / Страна / Язык / Город	col1, col2 = st.columns(2)	st.columns(2, gap="medium")
Year / Face	col3, col4 = st.columns(2)	st.columns(2, gap="medium")

2.5. Вкладка «Статьи»
При просмотре статьи — Meta Title / Meta Description:

Элемент	Сейчас	Нужно
Meta Title / Description	col1, col2 = st.columns(2)	st.columns(2, gap="medium")

2.6. Вкладка «Промпты»
Параметры модели (Temperature, Freq Penalty, Pres Penalty, Top P):

Элемент	Сейчас	Нужно
Чекбоксы (4 шт)	col_t1, col_t2, col_t3, col_t4 = st.columns(4)	st.columns(4, gap="small")
Number inputs (4 шт)	col_v1, col_v2, col_v3, col_v4 = st.columns(4)	st.columns(4, gap="small")

4. Порядок внедрения

1.	Открыть файл frontend/app.py
2.	Сразу после строки st.set_page_config(...) вставить блок st.markdown(<style>...) с глобальными CSS-правилами из раздела 2.1
3.	Пройти по каждой вкладке и внести точечные правки из разделов 2.2 – 2.6 (замена columns, subheader, text_input)
4.	Перебилдить контейнер: docker compose up -d --build frontend
5.	Проверить визуально каждую вкладку в браузере (localhost:8501)

Время на внедрение: 30–45 минут. Все правки — в одном файле, бэкенд не затрагивается.

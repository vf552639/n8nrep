## Контекст

Продолжаем работу над системой массовой SEO-генерации текстов. Нужно внести три изменения в N8N workflow: добавить переменные в промпты и реализовать режим тестирования с точкой остановки.

---

## Задача 1: Добавить переменную с заголовками H1-H6 в промпт «Анализ конкурентов»

**Что сейчас:** в промпте используется только `{{ $('Сбор структур конкурентов в 1').first().json.h.flat().join('.') }}` — это структуры (вероятно, H-теги из URL-структуры или что-то похожее).

**Что нужно:** добавить вторую переменную, которая подставляет собранные после парсинга заголовки H1-H6 со страниц конкурентов.

**Как сделать:**
- Найти ноду, которая парсит страницы конкурентов и извлекает H1-H6 (или создать такую, если её нет).
- Убедиться, что результат парсинга H1-H6 агрегируется в единый JSON (аналогично тому, как работает нода «Сбор структур конкурентов в 1»).
- В промпте «Анализ конкурентов» добавить новую переменную, например: `{{ $('Сбор H1-H6 конкурентов').first().json.headings }}` (точное имя ноды и поля — по факту реализации).
- Формат передачи в промпт: строка с разделителем (например, `\n`) или структурированный список — чтобы LLM видел, какие заголовки используют конкуренты.

## Задача 2: Добавить переменную «среднее количество слов» в промпт «Первичная генерация»

**Что нужно:** в промпте «Первичная генерация» должна быть доступна переменная со средним количеством слов в текстах конкурентов.

**Как сделать:**
- В промпт «Первичная генерация» добавить переменную: `{{ $('Среднее количество слов в тексте и объединение текста в 1 json').first().json.average_words }}`
- Вставить её в тело промпта в контексте инструкции для LLM, например: «Целевой объём текста: около {{ ... }} слов» — чтобы модель ориентировалась на этот объём при генерации.

## Задача 3: Режим тестирования — точка остановки после первичной генерации

**Цель:** после первичной генерации текста workflow должен останавливаться и отдавать результат для ручной проверки. Запуск дальнейших шагов — только после подтверждения.

**Как сделать:**

1. **Endpoint для просмотра результата.** После ноды «Первичная генерация» добавить Webhook-ноду (или Respond to Webhook), которая возвращает результат генерации в JSON. Это позволит вызвать GET/POST-запрос и получить текст для ревью.

2. **Остановка workflow.** После отправки результата на endpoint workflow должен ждать. Варианты реализации (выбрать подходящий):
   - **Wait-нода** с внешним триггером (Webhook Resume URL) — workflow встаёт на паузу, продолжается только при вызове resume-URL.
   - **Два отдельных workflow:** первый генерирует и сохраняет результат, второй запускается вручную и продолжает обработку.

3. **Флаг тестового режима.** Добавить переменную окружения или параметр в workflow (например, `TEST_MODE = true`). Логика:
   - Если `TEST_MODE = true` → workflow останавливается после первичной генерации, ждёт подтверждения.
   - Если `TEST_MODE = false` → workflow идёт дальше без остановки (будущее поведение после окончания тестирования).

4. **Реализация через IF-ноду:** после «Первичная генерации» ставится IF-нода, которая проверяет флаг. True-ветка → Wait/Webhook (пауза). False-ветка → следующий шаг workflow.

**Результат:** я могу вызвать endpoint, посмотреть сгенерированный текст, и если всё ок — отправить запрос на продолжение. Когда тестирование завершится — переключаем `TEST_MODE` в `false`, и IF-нода перестаёт отправлять в ветку ожидания.
